ERRORS

If an error happens, the endpoints will send a JSON.

  • Returns:
    • JSON:
      • err: {The error message varies.}

GET: /

This is the home index endpoint. It doesn't do much except returning a Never Gonna Give You Up by Rick Astley lyrics.

GET: /robots

This is another endpoint that will send a JSON message as follows:

{"msg": "Please don't do anything bad to this service :)."}

Usually it's for bots or web-scrapers for instructions of which stuffs are not allowed to be taken.

POST: /v1/public/register

Register Endpoint. It will create User data.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • role
      • name
      • email
      • password
  • Returns:
    • JSON:
      • msg: User registered successfully.

POST: /v1/public/login

Login endpoint.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • email
      • password
      • remember
  • Returns:
    • JSON with Keys as follows:
      • token
      • userEmail
      • userId
      • userName
      • userRole

  • NOTE:

    • Protected API groups need "token" and "userId" on the Header. So from here, you will have to provide "token" and "userId" on the Header
    • :id or :replyId means that a number (for example, /v1/protected/issue/show/1) is stored as a variable with the variable's name as "id" and "replyId". Hence, in the example, the "id" has a value of "1".

PATCH: /v1/protected/user/:id/change-password

Change Password endpoint.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • old-password
      • new-password
      • confirm-password
  • Returns:
    • JSON:
      • msg: User registered successfully.

GET: /v1/protected/user/:id

Change Password endpoint.

  • Returns:
    • PLEASE ASK THE DEVELOPER FOR THE RETURN VALUE.

POST: /v1/protected/issue/create

Create Issue endpoint. Create an Issue and store the data into the database.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • title
      • description
      • severity
  • Returns:
    • JSON with Keys as follows:
      • issueID
      • msg: Data succesfully created.

GET: /v1/protected/issue/index

Create Issue endpoint. Shows all Issues available.

  • Returns:
    • PLEASE ASK THE DEVELOPER FOR THE RETURN VALUE.

GET: /v1/protected/issue/show/:id

Show Issue endpoint. Shows all Issues available.

  • Returns:
    • PLEASE ASK THE DEVELOPER FOR THE RETURN VALUE.

PATCH: /v1/protected/issue/update/:id

Update Issue endpoint. Update an Issue and store the data into the database.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • title
      • description
      • status
      • severity
  • Returns:
    • JSON with Keys as follows:
      • issueID
      • msg: Data has been updated succesfully.

DELETE: /v1/protected/issue/delete/:id

Delete Issue endpoint. Delete (softly) an Issue.

  • Returns:
    • JSON with Keys as follows:
      • msg: issue is deleted successfully

POST: /v1/protected/issue/show/:id/reply

Create Reply endpoint. Create a Reply and store into database.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • description
  • Returns:
    • JSON with Keys as follows:
      • replyID
      • msg: Reply added successfully!

PATCH: /v1/protected/issue/show/:id/update-reply/:replyId

Update Reply endpoint. Update a Reply and store into database.

  • Requires:
    • Form data which input names as follow: (case sensitive)
      • description
  • Returns:
    • JSON with Keys as follows:
      • msg: Data successfully updated.

DELETE: /v1/protected/issue/show/:id/delete-reply/:replyId

Delete Reply endpoint. Delete (softly) a Reply.

  • Returns:
    • JSON with Keys as follows:
      • msg: Data successfully deleted.
